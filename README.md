# Сортировщик фото шкафов НКУ (Windows, HEIC + OCR)

Готовый локальный проект под ваш сценарий:
- вход: пачка фото шкафов (`.heic/.jpg/.jpeg/.png`);
- поиск фото шильдика и извлечение **заводского номера**;
- сопоставление в Excel (лист `Итого`, колонки `Заводской №` и `№заказа`);
- поиск папки объекта в каталоге `Объекты` по номеру заказа;
- группировка: от шильдика до следующего шильдика;
- копирование/перемещение в целевую структуру;
- подробный CSV-лог.

---

## 1) Что умеет программа

1. Читает форматы `.heic/.jpg/.jpeg/.png`.
2. Сортирует фото по порядку съемки:
   - сначала EXIF дата/время;
   - если EXIF нет — время файла;
   - затем имя файла.
3. Для каждого фото запускает OCR в 4 ориентациях (0/90/180/270).
4. OCR работает по полному кадру, крупным областям и угловым областям.
5. Определяет, что фото похоже на шильдик, только при наличии нескольких характерных признаков.
6. Ищет заводской номер рядом с `завод. № / зав. № / заводской №`.
7. Сверяет номер с Excel-реестром.
8. Ищет папку объекта в корне `Объекты` по правилу начала имени:
   - `^<номер>(пробел|дефис|конец)`.
9. При неоднозначном объекте не выбирает случайно.
10. Ведет детальный `process_log.csv`.

## 3) Установка на Windows 10/11

> Ни интернет, ни облачные API при работе программы не требуются, всё локально.

### Шаг 1. Установите Python
- Установите Python 3.11 (рекомендуется).
- При установке включите галочку **Add Python to PATH**.

### Шаг 2. (Важно для OCR) Установите Microsoft Visual C++ Redistributable
- Обычно уже установлен. Если `easyocr/torch` не запускается, установите пакет VC++ 2015-2022 x64.

### Шаг 3. Установите зависимости
Откройте `cmd` в папке проекта:

```bat
python -m venv .venv
.venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

> `easyocr` при первом запуске может загрузить модели OCR. После этого работа идет локально.

## 4) Подготовка папок

Пример:
- входные фото: `C:\PHOTO_SORT\photos_in`
- выход: `C:\PHOTO_SORT\sorted`
- Excel: `C:\PHOTO_SORT\registry.xlsx`
- корень объектов: `Z:\СпецЭнергоБлок\Объекты\!В работе`

Программа автоматически создает в выходной папке:
- `_НЕ_РАСПОЗНАНО`
- `_НЕТ_В_РЕЕСТРЕ`
- `_НЕОДНОЗНАЧНО_ОБЪЕКТ`

## 5) Запуск GUI (рекомендуется)

```bat
python run_gui.py
```

или двойной клик `run_gui.bat`.

В GUI:
1. Выберите папку с фото.
2. Выберите Excel-файл реестра.
3. Выберите корень `Объекты`.
4. Выберите выходную папку.
5. Выберите режим `copy` или `move`.
6. Нажмите **Запустить сортировку**.

## 6) Запуск CLI

```bat
python main.py ^
  --photos-dir "C:\PHOTO_SORT\photos_in" ^
  --excel-file "C:\PHOTO_SORT\registry.xlsx" ^
  --objects-root "Z:\СпецЭнергоБлок\Объекты\!В работе" ^
  --output-dir "C:\PHOTO_SORT\sorted" ^
  --sheet-name "Итого" ^
  --serial-column "Заводской №" ^
  --order-column "№заказа" ^
  --copy
```

### Опции CLI
- `--copy` / `--move` — режим работы с файлами (по умолчанию copy)
- `--sheet-name` (по умолчанию `Итого`)
- `--serial-column` (по умолчанию `Заводской №`)
- `--order-column` (по умолчанию `№заказа`)
- `--serial-subfolder` — дополнительно создавать подпапку по заводскому номеру
- `--skip-before-first-label` — пропускать фото до первого шильдика (по умолчанию такие фото идут в `_НЕ_РАСПОЗНАНО`)
- `--no-remember-processed` — отключить файл состояния обработанных файлов
- `--no-qr` — отключить попытку чтения QR-кода

## 7) Логика распределения и безопасные сценарии

### 7.1 Если заводской номер не прочитан надежно
Группа уходит в `_НЕ_РАСПОЗНАНО`.

### 7.2 Если номер прочитан, но его нет в Excel
Группа уходит в `_НЕТ_В_РЕЕСТРЕ`.

### 7.3 Если папка объекта не найдена
Группа идет в папку с именем номера заказа в выходной директории.

### 7.4 Если найдено несколько папок объекта
Безопасный режим: группа идет в `_НЕОДНОЗНАЧНО_ОБЪЕКТ\<№заказа>`, в лог пишется неоднозначность.

## 8) Файл лога

`<output_dir>\process_log.csv` содержит поля:
- дата/время обработки
- имя файла
- определен ли шильдик
- распознанный заводской номер
- найден ли номер в реестре
- найденный №заказа
- найдено ли имя папки объекта
- итоговая папка назначения
- статус
- комментарий / текст ошибки

Статусы:
- `OK`
- `SERIAL_NOT_RECOGNIZED`
- `SERIAL_NOT_IN_REGISTRY`
- `OBJECT_FOLDER_NOT_FOUND`
- `OBJECT_FOLDER_AMBIGUOUS`
- `FILE_ERROR`

## 9) Ограничения и где OCR может ошибаться

1. Очень размытые фото, сильные блики, грязь/пленка на шильдике могут снизить точность OCR.
2. Если на обычном фото много табличек и цифр, возможны ложные срабатывания (минимизируется ключевыми признаками шильдика).
3. Если формат заводского номера в реальности отличается от шаблона цифр (6–12 знаков), нужно скорректировать regex в `ocr_engine.py`.
4. Если в Excel встречаются дубли одинакового заводского номера с разными заказами, сработает последнее значение из таблицы.
5. `easyocr` тяжелый по зависимостям; на слабом ПК первый запуск может быть долгим.

## 10) Как донастроить под ваши реальные шильдики

Откройте `photo_sorter/config.py`:
- `label_keywords` — словарь признаков шильдика;
- `region_specs` — набор зон OCR (можно усилить углы/боковые области);
- в `photo_sorter/ocr_engine.py`:
  - `serial_number_pattern` — формат заводского номера;
  - `serial_hint` — шаблоны слов около номера.

Рекомендуется собрать 50–100 реальных фото и подстроить ключевые слова/regex под ваш фактический вид шильдика.
